What is novel here (vs. the reviewed manuscript)

1 · Correct angle definitions.

The manuscript's Eq. (1) computes the knee angle (hip–knee–ankle) but labels it "trunk-to-thigh". This pipeline computes both, each correctly named, and derives the true trunk-to-thigh (hip-flexion) angle from the shoulder keypoint — which 3-joint hip/knee/ankle tracking physically cannot do.

2 · Real motion-capture reference.

Validation is against the dataset's own OptiTrack-derived 2D ground truth (p2d.txt), not subjective observation. (The manuscript listed "absence of gold-standard motion capture" as a limitation — yet the dataset ships exactly that.)

3 · Automatic leg-side resolution.

Side is read per (subject, exercise) from dataset_2d.json (ex 03 = left, ex 07 = right), so the model measures the same limb the ground truth tracks — removing a large source of apparent error.

4 · Clusterable per-session reporting.

The pipeline emits per-session aggregates so downstream stats can cluster by subject/session, avoiding the frame-level pseudoreplication that inflates significance.
